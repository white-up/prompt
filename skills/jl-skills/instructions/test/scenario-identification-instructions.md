# 场景识别指令 (Scenario Identification Instructions)

此文档指导 Agent 执行场景测试生成的**场景识别（阶段 2）**。

> [!IMPORTANT]
> **目标**: 识别全面的测试场景，包括正常、异常和边界情况。
> **原则**: 覆盖率优先，先广度后深度。

## 步骤 1: 正常流程 (Happy Path) - P0

1.  **识别**:
    - 主流程成功路径。
    - 各分支的正常完成路径。
2.  **定义**:
    - ID, 名称, 前置条件, 步骤, 预期结果。
3.  **交互**:
    - 输出列表并确认。

## 步骤 2: 异常流程 (Exception) - P1

1.  **业务异常**: 规则校验失败、状态不符。
2.  **系统异常**: 数据库失败、网络超时。
3.  **补偿/回滚**: 事务失败后的处理。
4.  **交互**:
    - 输出列表并确认。

## 步骤 3: 边界测试 (Boundary) - P1

1.  **输入边界**: 最大/最小值、长度、空值。
2.  **业务边界**: 金额限制、时间窗口、库存阈值。
3.  **交互**:
    - 输出列表并确认。

## 步骤 4: 并发与幂等 (Advanced) - P2

1.  **并发**: 资源竞争、锁机制。
2.  **幂等**: 重复请求的处理。
3.  **交互**:
    - 输出列表并确认。

## 步骤 5: 汇总与保存

1.  **统计**:
    - 按优先级汇总数量。
2.  **确认**:
    - 最终确认场景清单。
3.  **保存**:
    - 暂存到状态文件。
